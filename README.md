# UDF_Python4Property

Fluent UDF for CO2 property tables generated by the Python4Property workflow.

## Files

- `CO2_propertiesV3_Python4Property.c`: nonuniform-grid Fluent UDF source file. This is the main version for Python4Property tables with local critical-region refinement.
- `CO2_propertiesV3_Python4Property_uniform.c`: fast uniform-grid Fluent UDF source file. Use only when both pressure and temperature grids are strictly uniform.
- `UDF可靠性分析.md`: design notes and reliability analysis.

## Required CSV Tables

Put these files in the Fluent working directory before running the import on-demand function:

- `co2_density.csv`
- `co2_viscosity.csv`
- `co2_conductivity.csv`
- `co2_cp.csv`
- `co2_phase.csv` optional, used only for phase-boundary diagnostics

The expected table format is:

```text
P(MPa)/T(K),220.0,220.5,221.0,...
0.05,value,value,value,...
0.15,value,value,value,...
```

Pressure in the first column must be MPa. Temperatures in the header must be K.

## Fluent Hooking

1. Compile `CO2_propertiesV3_Python4Property.c` in Fluent for nonuniform tables, or compile `CO2_propertiesV3_Python4Property_uniform.c` for strictly uniform tables.
2. Run the on-demand function `CO2_import_property_tables`.
3. Hook material properties:
   - Density: `CO2_table_density`
   - Viscosity: `CO2_table_viscosity`
   - Thermal conductivity: `CO2_table_thermal_conductivity`
   - Specific heat: `CO2_table_cp`
4. Optionally run `CO2_test_property_tables` to print one test point.

For the uniform-grid version, use these hook names instead:

- On-demand import: `CO2_import_property_tables_uniform`
- Density: `CO2_uniform_density`
- Viscosity: `CO2_uniform_viscosity`
- Thermal conductivity: `CO2_uniform_thermal_conductivity`
- Specific heat: `CO2_uniform_cp`
- Test function: `CO2_test_property_tables_uniform`

## Cp Treatment

`DEFINE_SPECIFIC_HEAT` in Fluent receives temperature but not local cell pressure. Therefore, Cp cannot be interpolated as a true pressure-temperature two-dimensional function through this macro.

This UDF still reads `co2_cp.csv`, but it extracts a one-dimensional `Cp(T)` curve from the pressure row nearest to:

```c
#define CP_REFERENCE_PRESSURE_PA 14000000.0
```

For the current sCO2-LBE jet conditions of roughly 12-16 MPa, 14 MPa is a reasonable default reference pressure. Change this macro before compilation if another operating pressure is more representative.

For speed in large Fluent cases, the enthalpy returned by `DEFINE_SPECIFIC_HEAT` currently uses the common Fluent UDF form:

```c
*h = cp * (T - Tref);
```

This is faster than integrating `Cp(T)` during every property call. If high-fidelity energy consistency becomes necessary later, the preferred upgrade is an enthalpy table `h(P,T)` rather than forcing pressure-dependent Cp into `DEFINE_SPECIFIC_HEAT`.

## Boundary and Phase Handling

- Density, viscosity and thermal conductivity use nonuniform bilinear interpolation in pressure-temperature space.
- Requests outside the table range are clamped to the nearest table boundary, and Fluent prints limited warnings.
- If `co2_phase.csv` is present, the UDF warns when a bilinear interpolation cell crosses a phase-family boundary. It does not automatically change interpolation formulas across the boundary.

The nonuniform and uniform UDFs include compile-time switches:

```c
#define ENABLE_PHASE_DIAGNOSTICS 0
#define ENABLE_RANGE_WARNINGS 0
```

Set these macros to `1` for debugging, then recompile the UDF. For long production runs, keep them at `0`. Phase diagnostics only print warnings and do not change interpolated property values, so disabling them improves speed without changing the numerical property values.

## Performance Notes

- CSV reading happens only when the on-demand import function is executed; it is not the main per-iteration cost.
- The nonuniform version uses binary search to find pressure and temperature intervals. This is flexible and suitable for critical-region refinement, but slower than direct indexing.
- The uniform version uses direct row/column indexing and is faster, but it requires strictly uniform pressure and temperature grids.
- The specific-heat enthalpy calculation uses `h = cp * (T - Tref)` for speed. This follows common Fluent UDF examples but is less thermodynamically rigorous than an integrated enthalpy or `h(P,T)` table.

## Important Limitations

- This UDF is a table-interpolation implementation, not an equation-of-state solver.
- Near the critical point and phase boundaries, interpolation reliability still depends strongly on table resolution and phase consistency.
- If pressure-dependent Cp or enthalpy is essential, a different Fluent energy-property strategy is needed, such as a pressure-temperature enthalpy table or a custom energy source/material formulation.
