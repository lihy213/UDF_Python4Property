# CO2_propertiesV3_Python4Property UDF 可靠性分析

## 1. Cp 是否适合改为二维表

从物理上讲，sCO2 的定压比热 `Cp` 是压力和温度共同决定的，特别是在临界点附近，`Cp(P,T)` 会出现尖峰。因此，用 CoolProp 生成二维 `Cp(P,T)` 表本身是合理的，也便于离线验证和后续更高级的能量模型使用。

但是，在 Fluent 常用的 `DEFINE_SPECIFIC_HEAT(name, T, Tref, h, yi)` 接口中，UDF 只能获得温度 `T` 和参考温度 `Tref`，不能获得当前单元压力。因此，直接把 `Cp` 做成二维表并不能在这个接口里真正调用局部压力。若强行使用二维表，只能选择一个代表压力、入口压力或固定参考压力对应的 `Cp(T)` 曲线，本质上仍然是温度函数。

因此，本版本采用折中策略：保留二维 `co2_cp.csv` 的读取能力，但从中提取最接近 `CP_REFERENCE_PRESSURE_PA` 的一行作为 `Cp(T)`。对于当前 12-16 MPa、350-400 degC 的 sCO2 入口条件，默认参考压力设为 14 MPa。这样做比旧 UDF 只读第一行更合理，也便于将来改参考压力。

需要注意的是，如果数值模拟中射流发生强烈膨胀，局部压力从 12-16 MPa 降到远低于参考压力，则 `Cp(T)` 会偏离真实 `Cp(P,T)`。这会影响能量方程、温度场和后续密度反馈。若论文中需要严肃讨论临界区热容尖峰或跨压力热力学路径，建议进一步开发压力-温度焓表 `h(P,T)` 或使用 Fluent 支持压力相关热物性的其他接口/模型策略。

## 2. 新 UDF 的代码结构

新 UDF 的核心结构分为五部分：

1. 表格数据结构：`Table2D` 保存非均匀温度网格、压力网格和二维物性值；`PhaseTable2D` 保存相态族。
2. 读表模块：读取 Python4Property 输出的 CSV 表头温度和第一列压力，不再假设固定 `0.5 K` 和 `0.1 MPa` 均匀步长。
3. 插值模块：密度、粘度和导热率使用非均匀网格双线性插值；超出表格范围时采用边界夹取并输出警告。
4. 相态诊断模块：读取 `co2_phase.csv`，当插值单元跨越相态族边界时输出警告，提示该区域的双线性插值可能不可靠。
5. Cp 与焓模块：从二维 Cp 表提取参考压力下的 `Cp(T)`，并用梯形积分计算 `h = integral Cp(T)dT`。

## 3. 与 sCO2-LBE 射流过程的适配性

对于 sCO2-LBE 射流，密度、粘度和导热率随压力和温度变化显著，尤其在膨胀、临界区和低温区域。新 UDF 读取非均匀二维表，能够适配此前 Python 代码中“全局粗网格 + 临界区加密”的物性生成策略，比旧 UDF 的固定均匀网格假设更适合宽范围射流计算。

相态诊断对射流膨胀尤其重要。局部压力和温度可能穿越气相、液相、超临界态或两相边界。双线性插值在同一相态内部通常较稳定，但如果四个角点跨越相边界，插值值可能只是数值平均，不一定有物理意义。新 UDF 不直接修正这一问题，但会在 Fluent 控制台给出警告，便于后处理判断哪些区域需要更细网格或单独建模。

## 4. 相比旧 UDF 的主要改进

- 不再硬编码 `N_T=2561`、`N_P=400`、`DDT=0.5`、`DDP=100000 Pa`。
- 压力列按 MPa 读取并自动转换为 Pa，适配新版 Python 输出。
- 支持非均匀网格，因此可读取临界区加密后的表。
- 密度、粘度、导热率均使用真实表头坐标插值。
- Cp 不再默认读取第一行，而是使用可配置参考压力。
- 焓不再简单使用 `Cp*(T-Tref)`，而是对 `Cp(T)` 积分。
- 增加边界夹取警告和相边界诊断。

## 5. 仍需注意的风险

- `DEFINE_SPECIFIC_HEAT` 不能表达真实的 `Cp(P,T)`，这是 Fluent 接口限制，不是读表代码本身的问题。
- 相边界附近目前只报警，不做相内插值切换或两相热力学处理。
- 超出表格范围时采用夹取而不是外推，数值上更稳，但会牺牲边界外物理准确性。
- 如果 Fluent 并行计算时各计算节点无法访问 CSV 文件，需要确认工作目录和文件分发方式。
- 当前代码以表格插值为目标，不负责判断 CoolProp 原始值是否跨入不可用区、固相区或非平衡两相区。

## 6. 建议的后续提升

1. 为论文验证阶段保留 `co2_phase.csv`，统计 Fluent 计算中相边界警告出现的位置和频率。
2. 对 Cp 进行敏感性测试，例如 12 MPa、14 MPa、16 MPa 三条参考压力曲线分别计算，评估温度场差异。
3. 若 Cp 敏感性很强，优先开发 `h(P,T)` 表，并围绕焓、温度反算和能量方程一致性重新设计 UDF。
4. 对临界区计算结果，不只看单点相对误差，还要看射流主路径是否穿过 Cp 尖峰区域。
5. 在 Fluent benchmark 中同步输出局部 `P-T` 散点，叠加到 CO2 相图和表格加密区域上，判断当前物性表是否覆盖真实射流路径。

## 7. 计算速度与宏开关注意事项

Fluent 中真正影响迭代速度的不是 CSV 读表过程，而是每个单元、每次迭代中反复调用的物性函数。对于 360 万网格、10 进程这类算例，密度、粘度、导热率和 Cp 都会被频繁调用，因此单次物性调用中的额外判断会被放大。

当前非均匀版本 `CO2_propertiesV3_Python4Property.c` 仍然是主力版本，适用于“全局网格 + 临界区加密”的 Python4Property 表。它为了支持非均匀网格，需要通过二分查找确定当前压力和温度所在区间，因此比旧版均匀网格 UDF 的直接行列索引慢一些。

为了兼顾调试和生产计算，正式版加入了两个编译期宏：

```c
#define ENABLE_PHASE_DIAGNOSTICS 0
#define ENABLE_RANGE_WARNINGS 0
```

建议设置方式如下：

- 调试阶段：可将 `ENABLE_PHASE_DIAGNOSTICS` 设为 `1`，检查插值单元是否跨越相态边界。
- 正式长时间计算：保持 `ENABLE_PHASE_DIAGNOSTICS=0` 和 `ENABLE_RANGE_WARNINGS=0`，减少每次物性调用中的判断和控制台输出。

需要强调的是，相态诊断目前只报警，不改变插值值。因此关闭相态诊断不会改变密度、粘度、导热率和 Cp 的数值结果，只会少输出警告并略微提高速度。

## 8. Cp 焓计算的当前策略

此前版本曾将 `DEFINE_SPECIFIC_HEAT` 中的焓写成对 `Cp(T)` 的分段积分。这个做法在热力学上更一致，但在 Fluent 中会被非常频繁地调用；如果每次调用都循环积分，可能明显拖慢大网格计算。

当前版本为了优先保证计算速度，采用 Fluent 官方示例中常见的形式：

```c
*h = cp * (T - Tref);
```

这与旧 UDF 写法一致，计算代价很低，适合作为第一阶段射流模拟和参数敏感性分析的默认方案。其局限是当 Cp 随温度或压力剧烈变化时，焓的一致性不如积分形式。后续如果发现温度场或能量收支对 Cp 特别敏感，建议不要继续在 `DEFINE_SPECIFIC_HEAT` 中强行处理二维 Cp，而是升级到压力-温度焓表 `h(P,T)` 或更完整的能量物性策略。

## 9. 均匀网格快速版

仓库中新增了 `CO2_propertiesV3_Python4Property_uniform.c`，用于严格均匀压力和温度网格的快速计算。它的运行时索引方式接近旧 UDF：

```c
it = floor((T - Tmin) / dT);
ip = floor((P - Pmin) / dP);
```

因此速度会比非均匀版本更接近旧版 UDF。但它不能用于临界区局部加密表。如果输入表不是均匀网格，该版本会提示使用非均匀 UDF。

推荐选择：

- 使用临界区加密表：选择 `CO2_propertiesV3_Python4Property.c`。
- 使用严格均匀表并追求速度：选择 `CO2_propertiesV3_Python4Property_uniform.c`。
- 学习代码逻辑：阅读本地的 `CO2_propertiesV3_Python4Property_comments.c`，不要将该详注版作为 GitHub 正式版上传。

## 10. 声速处理

对于可压缩 sCO2 射流，声速直接影响马赫数、膨胀波、激波和欠膨胀射流结构判断。当前 UDF 已支持读取 `co2_sound_speed.csv`，并提供可挂接的声速物性函数：

- 非均匀版本：`CO2_table_sound_speed`
- 均匀版本：`CO2_uniform_sound_speed`

这里推荐直接使用 CoolProp 生成的声速表，而不是在 UDF 中仅由密度表数值求导得到声速。原因是热力学声速本质上满足：

```text
a^2 = (∂p/∂rho)_s
```

即等熵压力-密度导数，而不是普通的等温密度斜率。Fluent 手册中的体积模量示例适合弱可压缩液体或人为定义的线性密度模型，其中声速由给定体积模量和密度模型解析得到。对于真实气体/超临界 CO2，声速同时受温度、压力、热容比、真实气体状态方程及近临界物性影响。若只对 `rho(P,T)` 表做简单压力导数，通常得到的是近似的等温压缩性信息，并不等价于等熵声速。

因此，当前更稳妥的策略是：

1. Python 使用 CoolProp `PropsSI("A", "T", T, "P", P, "CO2")` 生成声速表。
2. UDF 像密度、粘度、导热率一样对 `a(P,T)` 进行二维插值。
3. 后处理中用 `Mach = |v| / a` 计算局部马赫数。

这种方法避免了在 Fluent UDF 中对密度表做噪声敏感的数值微分，也保持了声速与 CoolProp 真实气体热力学模型的一致性。
